# Contributing

Thanks for contributing to AegisLab.

## Workflow

1. Open or reference an issue.
2. Keep changes focused and testable.
3. Attach validation evidence in the pull request.
4. Document behavior changes clearly.

## Documentation Contributions

When updating docs, verify internal links and keep concept descriptions aligned with current implementation.
